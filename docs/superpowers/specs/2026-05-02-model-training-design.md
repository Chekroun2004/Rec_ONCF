# Model Training Design — Rec_ONCF

**Date:** 2026-05-02  
**Status:** Approved  
**Scope:** Phase ML — training pipeline (O/D prediction model)  
**Deferred:** ONCF schedule API integration (last phase)

---

## 1. Objective

Train an XGBoost multiclass model that predicts the most likely `LiaisonId` (origin→destination pair) a user will book next, based on their booking history and contextual features.

The model powers a **zero-click search** experience: the top-1 recommendation is pre-filled on the ONCF app home screen before the user types anything.

**Primary metric:** `hit_rate@1` — the true liaison is the model's top prediction.  
**Secondary metrics:** `hit_rate@3`, `mrr@3`.

---

## 2. Constraints

| Constraint | Value |
|---|---|
| Cold-start threshold | `nb_voyages_total < 3` → no recommendation, show search bar |
| Max candidates | 10 (generated by `candidates.py`) |
| Max k | 3 |
| No GPS data | Never |
| Privacy | Loi 09-08 / CNDP — `CodeClient` excluded from X |

## 5. Model Architecture

```
sklearn Pipeline
├── ColumnTransformer
│   ├── OneHotEncoder(handle_unknown="ignore")  → categorical columns
│   └── passthrough                              → numeric columns
└── XGBClassifier
    ├── objective        = "multi:softprob"
    ├── eval_metric      = "mlogloss"
    ├── tree_method      = "hist"          ← native NaN handling (no imputer needed)
    ├── n_estimators     = 300
    ├── learning_rate    = 0.08
    ├── max_depth        = 8
    ├── subsample        = 0.9
    ├── colsample_bytree = 0.8
    ├── reg_lambda       = 1.0
    └── n_jobs           = -1
```

**Why `tree_method="hist"`:**  
Handles `NaN` natively in `PrixParLiaison` (697 nulls) and `days_since_prev` (69 449 nulls — first trip per client). No `SimpleImputer` needed, cleaner pipeline.

---

## 6. Train/Test Split

Temporal split — no random shuffle, no data leakage:

```python
df_sorted = df.sort_values("DateHeureDepartVoyageSegment")
cut = int(len(df) * 0.8)
train = df_sorted.iloc[:cut]   # ~393 344 rows — older bookings
test  = df_sorted.iloc[cut:]   # ~98 336 rows  — most recent bookings
```

## 7. Evaluation Metrics

Computed on the test set and written to `reports/offline_metrics.json`.

| Metric | Description | Priority |
|---|---|---|
| `hit_rate@1` | True liaison is the #1 prediction | **Primary** |
| `hit_rate@3` | True liaison is in top-3 predictions | Secondary |
| `mrr@3` | Mean Reciprocal Rank at 3 | Secondary |

`mrr_at_k` added to `src/rec_oncf/metrics.py`.

---

## 9. Training Script

`scripts/03_train_ranker.py` — pure orchestrator, no business logic:

```
1. Load data/processed/features.parquet
2. temporal_split(train_frac=0.8)
3. train_xgb_multiclass(df_train, label_col="LiaisonId", time_col="DateHeureDepartVoyageSegment")
4. predict_proba(artifacts, df_test)
5. Compute hit@1, hit@3, mrr@3
6. Write reports/offline_metrics.json
7. save_artifacts → models/
```
